3. THE PROCESS (7 STEPS - 15 MINUTES)
─────────────────────────────────────────────────────────────────────────────

Step 1: Structure & Scope (2 min)
  → What data do I have? When? How much severity?

Step 2: Operation Tracing (2 min)
  → Follow each user action from start to finish

Step 3: Error Identification (2 min)
  → Find ALL errors, categorize them

Step 4: Failure Point (2 min)
  → Exactly WHEN did things go wrong?

Step 5: Root Cause Analysis (2 min)
  → Why did it go wrong? (Ask "Why?" 3+ times)

Step 6: Impact Assessment (2 min)
  → How bad? Who affected? What's the severity?

Step 7: Expected vs Actual (2 min)
  → What should have happened vs what did?

Step 8: Output Report (1 min)
  → Document findings in required format

─────────────────────────────────────────────────────────────────────────────
4. QUICK REFERENCE
─────────────────────────────────────────────────────────────────────────────

KEY PRINCIPLES:
  ✅ Ground ALL conclusions in log evidence
  ✅ Distinguish symptoms from root causes
  ✅ Ask "Why?" until you reach the fundamental issue
  ✅ Create expected vs actual comparison
  ✅ Propose testable verification steps

MUST NOT:
  ❌ Speculate without evidence
  ❌ Stop at symptoms (must find root cause)
  ❌ Skip "what safeguard was missing?"
  ❌ Propose fixes before understanding cause

5. THE OUTPUT FORMAT YOU'LL PRODUCE
─────────────────────────────────────────────────────────────────────────────

Your analysis must include:

  ✓ Executive Summary (1-2 sentences)
  ✓ Session Overview (duration, operations, status)
  ✓ Critical Issues (for each: severity, time, message, cause, impact)
  ✓ Expected vs Actual (comparison table)
  ✓ Root Cause Analysis (why it happened)
  ✓ Recommendations (prioritized by severity)
  ✓ Verification Steps (how to test the fix)
  ✓ Log Evidence (relevant excerpts with timestamps)
